# 模型更新策略（Model Updates）

本项目允许引入模型，但原则是：**模型是证据源，不是最终裁决者**。

## 1) 分离“证据”与“裁决”

推荐模式：

- 模型输出写入 `EvidenceStore` 新字段（例如 `llm_semantic`）
- 评分/规则层引用该字段，但保留可解释的贡献分解
- 当模型不可用/超时时，不应导致系统整体不可用；应降级并记录

## 2) 版本化与评估

对模型更新建议：

- 明确 `model_version` 并随事件输出记录
- 在固定 benchmark 上对比新旧版本（见 `evaluation/`）
- 小流量灰度发布，观察误报/漏报与运行时指标

## 3) 回滚与审计

- 保持快速回滚（切换配置或禁用模型证据源）
- 结合 JSONL 录制与回放，对比更新前后差异并定位原因

仓库的 `models/lightweight_classifier/` 当前为占位实现，可按上述策略接入。
